# Nutrition Tracker App — Competitor Screenshot Analysis

## Overview
This appears to be a **meal/calorie tracking app** (possibly Lifesum or a similar nutrition app). The screenshot shows a **daily meal log** with macro breakdowns per meal, organized by time-based meal categories.

---

## Color Usage
- **Soft pink/rose**: Dominant color (`~#F8D7E0`) used for the macro summary header area — creates a calming, health-oriented, feminine aesthetic.
- **Dark green**: Deep forest green (`~#1B4332`) used for icons (three-dot menu buttons), the active tab pill, and the circular progress ring — acts as the primary action/brand color.
- **White**: Clean white cards for individual meal entries — creates clear separation and readability.
- **Light gray background**: Subtle off-white/light gray (`~#F5F5F5`) background for the overall screen — non-intrusive base.
- **Macro color coding**: Subtle pastel color bars next to each macro value:
  - **Green** accent bar for Protein
  - **Yellow** accent bar for Fat
  - **Pink/magenta** accent bar for Carbs
- **Minimal color palette overall**: Only 3-4 colors used throughout — intentionally restrained and calming.

## Typography
- **Date header**: Medium-weight, dark text ("Monday, 31 August") with a calendar icon — clean, informational.
- **Macro numbers**: Large, bold, dark sans-serif numbers (87, 103, 167) — high visual priority for at-a-glance scanning.
- **Macro labels**: Small, light gray text beneath the numbers ("Protein", "Fat", "Carbs") — secondary importance.
- **Calorie count**: Large bold number (1566) inside the circular progress ring with "kcal" label below — the centerpiece metric.
- **Meal tab labels**: Medium weight with time ranges shown in parentheses ("Breakfast (00:00-12:00)") — functional but clear.
- **Food item names**: Bold, dark, medium-sized ("Salad with egg", "Vegetables") — easy to scan.
- **Nutritional details**: Smaller gray text for calorie/weight info ("🔥 294 kcal • 100 g").
- **Individual macros in cards**: Bold numbers (25g, 21g, 14g) with small labels — scannable grid format.

## Layout & Spacing
- **Three-zone layout**:
  1. **Top zone**: Date + avatar (compact header)
  2. **Middle zone**: Macro summary bar with circular progress (pink background)
  3. **Bottom zone**: Meal tabs + scrollable food item cards
- **Macro summary bar**: Horizontal row of 3 macro values + circular kcal ring on the right. Well-balanced, no wasted space.
- **Meal tabs**: Horizontal scrollable tab bar with 4 categories (Breakfast, Lunch, Dinner, Snacks). Active tab has dark pill background with white text.
- **Food item cards**: Full-width, stacked vertically. Each card contains:
  - Left: Food emoji/illustration (~40px)
  - Center: Name + calorie info
  - Right: Three-dot menu icon
  - Bottom row: Three macro values with color-coded accent bars
- **Card internal layout**: Clean 2-row structure (top: name/calories, bottom: macro grid).
- **Bottom nav**: 5-tab icon-only navigation (Home, Search, List, Favorites, Cart).

## Design Patterns
- **Circular progress ring**: Shows daily calorie progress — green arc on light gray track. Classic fitness/health app pattern (similar to Apple Health rings).
- **Color-coded macro bars**: Tiny vertical accent bars next to each macro value provide instant visual categorization without overwhelming the design.
- **Time-based meal categories**: Meal tabs include time ranges — adds structure and auto-categorization context.
- **Food emoji/illustrations**: Each food item has a small illustrated icon — makes the list visually engaging and scannable.
- **Three-dot menu**: Action overflow on each food card — allows edit/delete without cluttering the main view.
- **Avatar in header**: Small user avatar in top-right — personalization without prominence.
- **Stacked card pattern**: Each food entry is its own card with consistent internal structure — highly scannable.
- **Macro grid pattern**: The 3-column macro display (Protein | Fat | Carbs) is repeated at both the summary level and per-food-item level — consistent mental model.

## Spacing & Sizing Observations
- **Header zone**: ~50px (date + avatar row).
- **Macro summary area**: ~80px height with the pink background.
- **Circular progress ring**: ~60px diameter.
- **Meal tabs height**: ~44px with comfortable padding.
- **Food item cards**: ~120px tall each with ~12px gap between cards.
- **Card corner radius**: ~16px — consistent, modern rounded feel.
- **Card padding**: ~16px internal padding on all sides.
- **Macro accent bars**: ~3px wide, ~20px tall — subtle but effective.
- **Bottom nav**: ~50px, icon-only (no labels visible in this state).

## Key Takeaways
1. **Calm, health-focused aesthetic** — soft pink + green palette is deliberately soothing, avoiding the aggressive feel of commerce apps.
2. **Data-dense but clean** — manages to show a lot of nutritional data without feeling cluttered, thanks to consistent card structures and whitespace.
3. **Consistent macro visualization** — the same Protein/Fat/Carbs pattern repeats at the summary and item level, building a strong mental model.
4. **Circular progress ring** is the emotional anchor — tells the user at a glance how they're doing today.
5. **Minimal use of color** — constraining to pink, green, yellow, and gray keeps the interface serene and focused.
6. **No promotional content** — pure utility. No ads, deals, or upsells visible. User-goal-centric design.
7. **Time-structured meals** create a natural daily narrative (Breakfast → Lunch → Dinner → Snacks).
8. **Food illustrations** add visual warmth to what could otherwise be a dry data interface.
